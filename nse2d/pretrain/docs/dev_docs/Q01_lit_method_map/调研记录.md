## 冲突与缺口

- 证据冲突：
  - staged / offline anchor 从项目内结果看最好，但它不是 strict data-free。
  - learnable GRF 与 self-training bootstrap 都能改善一些局部指标，但没有解决“multi-step 越滚越坏”。
- 关键缺口：
  - 缺外部 literature、benchmark 和 baseline 的系统对照。
  - 缺正式的 `data-free` 边界定义与 anchor 分类。
  - 缺一版稳定的“挑战 -> 后续问题”映射。
- 还需要补什么：
  - 相关工作关键词、代表论文、适用任务边界。
  - 不同方法族的比较维度：是否 strict data-free、是否需要真实状态锚点、是否改善长期 rollout。
  - 各 major challenge 与 `Q02-Q06` 的归属规则。

## 当前结论

- 可直接采用：
  - Q01 当前可先把方法图谱分为五类：reference anchored baseline、fixed random prior / PDE-only、learnable prior / learnable GRF、bootstrap / self-training、小数据生成模型边界路线。
  - 当前 major challenges 至少包括：`data-free` 口径未定、随机态物理性不足、真实物理流形分布错配、one-step 与 rollout 脱钩、五场联合结构不足、缺外部证据。
  - `batch_size`、why staged best、one-step vs rollout、爆炸 pattern 可以先作为后续拆分候选，而不是在 Q01 内直接定论。
- 暂不能下结论：
  - 哪个方法族最终最有希望把 strict data-free 跑通。
  - 外部相关工作里是否已经有人解决了最接近本项目设定的问题。
  - 当前爆炸 pattern 的第一主因到底是分布错配、误差传播，还是少数字段 / 模态失衡。

## 候选方法图谱

| 方法族 | 当前定义 | 与 strict data-free 的关系 | 项目内当前判断 | 建议后续去向 |
|--------|----------|----------------------------|----------------|--------------|
| reference anchored baseline | `GRF` 预训练后再接 simulation data，或其他带真实状态锚点的路线 | 不是 strict data-free，但必须保留为参照系 | 目前经验最强 | 保留在 Q01 作为 reference |
| fixed random prior / PDE-only | 固定随机态先验，主要靠 PDE loss 训练 | 最干净的 strict data-free 对照 | 重要但上限未明 | `Q03` |
| learnable prior / learnable GRF | 端到端优化输入分布或其参数化 | 试图把随机态推近物理流形 | 有局部改善，但长期效果未证成 | `Q04` |
| bootstrap / self-training | 用 teacher 逐步生成或刷新训练态 | 当前最积极的纯 online 路线 | 现阶段最值得继续跟踪 | `Q05` |
| small-data generative model | 用少量真实数据学习分布后再生成训练态 | 相对 strict data-free 更像边界 / fallback 路线 | 有研究价值，但当前不是主攻 | `Q06` |
