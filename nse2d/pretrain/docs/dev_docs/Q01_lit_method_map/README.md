## 1. 问题定义

- 这个问题具体是什么：
  - 回答四件事：相关文献到哪里了；当前已有方法有哪些主要分支；strict data-free 训练面临哪些核心挑战；后续应把哪些问题拆成独立主问题继续跟。
- 为什么重要：
  - 这是整个问题树的上游入口。如果这层不先收敛，后续很容易只在本地实验空间里反复堆 trick，而不清楚“别人做到哪、我们真正缺哪一块”。
- 当前不处理什么：
  - 不把项目内观察误写成外部文献事实。
  - 不代替后续 `Q02-Q06` 的路线验证、理论分析和实验记录。
  - 不修改 `main.md`、`shared_context/` 或旧 `ROOT_物理随机态生成与彻底data_free训练/` 文档。

## 2. 当前结论

- 已确认：
  - 当前任务语境是 2D 5-field MHD surrogate training。
  - 项目内最好的经验路线仍是“GRF 预训练 + simulation data 继续训练”，说明真实状态锚点仍关键。
  - 当前 raw GRF / raw online 路线里，常见现象是 `train loss` 和单步指标变好，但多步 rollout 仍变差。
  - 对 Q01 而言，当前最缺的不是再加一个训练 trick，而是系统化的方法图谱、挑战图谱与文献支撑下的子问题树。
- 仍不确定：
  - 与本问题最直接相关的外部文献、benchmark 和可复用 baseline 还没有系统补齐。
  - `data-free` 的边界定义与 anchor 分类还未正式定稿。
  - `batch_size`、why staged best、one-step vs rollout、爆炸 pattern 分别应该归到上游共性问题还是 route-specific 子问题，仍需继续定边界。
