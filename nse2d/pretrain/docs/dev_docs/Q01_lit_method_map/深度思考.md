## 已确认事实

- 事实 1：当前最好的项目内经验路线仍带真实状态锚点，因此 strict data-free 还未成立。
- 事实 2：raw GRF / raw online 路线里，常见现象是 one-step 指标改善，但 multi-step rollout 仍变差。
- 事实 3：当前已有候选路线至少包括 fixed random prior / PDE-only、learnable prior / GRF、bootstrap / self-training，以及小数据生成模型边界路线。

## 核心假设

- 假设 A：当前主矛盾首先是训练态分布与真实物理流形错位，而不只是某个 loss 还不够强。
- 假设 B：Q01 应先做上游文献 / 方法 / challenge map，不应在这里直接宣判哪条路线已经胜出。
- 假设 C：route comparison 需要和“后续问题归属”一起确定，否则文档边界会继续混写。

## 反例攻击

- 最脆弱的前提：
  - 把当前项目内观察直接当成外部 literature 结论。
- 什么现象一旦出现就说明这条路不通：
  - 如果补完 literature 后发现当前方法图谱的主分类并不成立，Q01 的方法树需要重写。
  - 如果后续证据表明 one-step vs rollout 的分裂只是协议问题，而不是 route-level challenge，当前 challenge tree 也需要调整。
- 目前最强反对意见：
  - 现有证据主要来自项目内实验与讨论，尚不足以支撑“已有方法做到哪里了”的稳结论。

## 路线比较

| 路线 | 为什么可能成立 | 最大风险 | 当前判断 |
|------|----------------|----------|----------|
| reference anchored baseline | 最接近真实训练态分布，是当前最强参考线 | 不是 strict data-free，不能直接回答根问题 | 只保留为参照系，不当作解答 Q01 的终点 |
| fixed random prior / PDE-only | 是最干净的 strict data-free 对照，能直接暴露随机态先验上限 | 随机态可能离真实物理流形过远，导致 rollout 长期不稳 | 必须保留，但当前不应假设它单独足够 |
| learnable prior / learnable GRF | 可能通过 joint optimization 把输入分布推近更有训练价值的区域 | 可能只改善局部指标，而无法改善长期 rollout | 保留为重要候选路线，但暂不因局部收益升格为主线 |
| bootstrap / self-training | teacher 有机会把训练态逐步推向更稳定、更物理的区域 | 也可能放大模型误差，把错误模式反复自举 | 当前最值得继续跟踪的 pure online 候选路线 |
| small-data generative model | 可以注入比独立随机态更强的分布先验 | 已经偏离 strict data-free，更像边界或 fallback 方向 | 有研究价值，但不作为本阶段主攻线 |
