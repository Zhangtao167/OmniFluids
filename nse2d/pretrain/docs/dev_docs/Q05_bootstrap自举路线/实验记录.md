# 实验记录

> 只记录 `Q05` 本路线的 bootstrap / self-training 相关实验，不再把这条线混在根问题总记录里。

## 元信息

- QID：Q05
- 最近更新时间：2026-03-11
- 本问题实验目标：
  - 判断 teacher 生成态是否比原始 `GRF` 更有训练价值。
  - 判断 `teacher` 上线时机、刷新周期、rollout 步数、teacher source 与 rollout curriculum 是否会改变稳定性。
  - 判断 `soft-Linf`、`multi-step PDE`、`relative PDE loss` 能否把 bootstrap 线推进到更稳的 rollout 区间。
  - 判断 dynamic target-RMS relative loss 与 fixed-scale relative loss 哪种归一化更利于 field balance 与 rollout。
  - 判断 `rollout_dt` 是否应与 `dt_data` 对齐，还是保留更细的 model substep 更稳。
  - 判断 multi-step rollout 是否应保留跨步梯度，还是在中间状态 `detach` 更稳。
  - 判断 `batch_size` 是否是当前 `Q05` 结论里的关键干扰变量。
  - 判断输入扰动 / ensemble RHS smoothing 这类 target-smoothing 方案是否值得开新分支。

## 实现改动

- `exp13`：
  - 历史 bootstrap 基线。
  - 晚启动：`start=100000`。
  - `teacher rollout=3`，`update_every=20000`，`batch_size=10`。
- `exp18`：
  - 把 `teacher` 提前到 `30000`。
  - 把 teacher rollout 从 `3` 增到 `5`。
  - 仍是纯 PDE loss，没有 `soft-Linf` 和 `multi-step PDE`。
- `exp22`：
  - 在 `exp18` 基础上加 `soft-Linf`。
  - 再加 `multi_step_pde_loss=1`、`multi_step_pde_n=3`。
  - 为了显存把 `batch_size` 从 `10` 降到 `2`，GPU 数也从 4 卡变成 2 卡。
- `exp23`：
  - 保留 `exp22` 的 loss 设计。
  - 把 `self_training_start` 改到 `100000`。
  - 把 `self_training_update_every` 从 `20000` 改到 `10000`。
- `exp24`：
  - 在 `exp22` 协议上加 `use_relative_loss=1`。
  - 实现说明写的是 `residual / target_rms_per_field.detach().clamp(min=1e-8)`。
  - 脚本文件名写 `MXE`，但 `exp_name` 写的是 `normed_mse`，命名上已有漂移。
  - 当前已在 `step_8000` 左右停止，原因是 early-to-mid rollout 明显恶化，未显示出可持续净收益。
- `exp26`：
  - 在 `exp22` 协议上保留 `use_relative_loss=1`，但把动态 `target RMS` 归一化改成固定 per-field scale。
  - 固定 scales 取自训练数据统计窗口 `[250, 300]`，顺序为 `n,U,vpar,psi,Ti`。
  - 核心目的是把“relative loss 是否有效”和“dynamic vs fixed scale 哪种更稳”拆开看。
  - 当前已在约 `step_2400` 手动停止，并把 `H800 : t1` 让给 `exp32`；因此现阶段只保留 early positive signal，不写后续窗口结论。
- `exp27`：
  - 在 `exp22` 协议上把 `rollout_dt` 从 `0.1` 改到 `1.0`。
  - 目标是直接测试“模型步长 = 数据步长”是否能减少内部 substep 误差积累。
  - 当前已落盘 `step_0`，但因为评估协议变成 `10 NFE`，还不能把首轮数值直接写成正结果。
- `exp28`：
  - 在 `exp22` 协议上只把 `multi_step_pde_detach` 从 `1` 改到 `0`。
  - 目标是做一个 clean no-detach ablation，看跨 rollout step 保留梯度是否更利于 bootstrap 稳定性。
  - 当前已落盘 `step_0`；但 no-detach 只影响训练反传路径，因此真正判断仍要看训练后的同阶段对照。
- `exp29`：
  - 在 `exp22` 协议上只把 `self_training_teacher_source` 从 latest 改成 `best`。
  - 目标是看 refresh 时用当前 best checkpoint，当 teacher source 是否比 latest training model 更稳。
  - 当前本地已有 run_tag，但这次尝试在初始评估窗口报 `No CUDA GPUs are available`；因此现阶段是 launch 问题，不是方法正负结果。
- `exp30`：
  - 在 `exp22` 协议上给 self-training rollout depth 加 curriculum。
  - 规则是 bootstrap 激活后先用 5-step teacher rollout，再每 `10000` step 增 1，最高到 6。
  - 当前只有用户口头同步 launch；本地结果目录与 log 仍待落盘。
- `exp31`：
  - 在 `exp22` 协议上只把 `soft_linf_weight` 从 `0.1` 改到 `0`。
  - 保留 `multi-step PDE`、self-training 时机与 refresh 协议、`batch_size=2`，作为 clean no-soft-Linf ablation。
  - 当前已由用户同步在 `20260308215522 : t2` launch；本地结果目录与 log 仍待落盘。
- `exp32`：
  - 在 `exp22` 协议上只把 `multi_step_pde_loss` 从 `1` 改到 `0`。
  - 保留 `soft-Linf`、self-training 时机与 refresh 协议、`batch_size=2`，作为 clean no-multi-step ablation。
  - 当前已由用户同步在 `H800 : t1` 所在 session 手动启动；terminal 头部已打印，但本地结果目录与首轮 eval 仍待落盘。
- 当前代码里已有但尚未单开 `Q05` 实验的相邻钩子：
  - `input_noise_scale`：训练时对输入加 Gaussian noise，target 保持不变。
  - 这还不是 ensemble RHS averaging / windowed perturb target；后者仍需要新实现。

## 关键结果

- 已回答结果：
  - `exp22` 是当前最强的 self-training / bootstrap 结果；当前汇总上，MHD `11.11`、GRF `9.614`，优于 `exp18` 的 `17.37 / 16.13` 与 `exp13` 的 `28.12 / 25.01`。
  - `exp22` 也优于当前 `learnable GRF` 主线：`exp20` 的 `22.21 / 22.67` 与 `exp21` 的 `25.72 / 27.49`。
  - 但 `exp22` 仍远差于 anchored baseline，例如 `exp_offline_pde (1GPU)` 是 MHD `0.4097`、GRF `1.060`，`exp1b (1GPU)` 是 MHD `0.4099`、GRF `1.030`。
  - 按 field 看，`exp22` 当前主要问题仍是 `U`。最终 inference 上，MHD 的 `U=19.65`、`n=15.74`、`vpar=9.31`；GRF 的 `U=26.70`、`vpar=8.03`，说明 rollout 爆炸并没有被根本解决。
- 部分回答结果：
  - 历史上“更晚启动”的 `exp13` 很差，而“更早启动”的 `exp18` 明显更强；但因为这两者还改了 teacher rollout 步数，所以只能说“更积极的 bootstrap 配置族更强”，还不能直接说“早启动一定更好”。
  - `exp23` 当前已经落盘到 `step_18000`，但距离 `step_100000` 激活 bootstrap 还很远；因此当前只能记录它的 pre-bootstrap 基线，而不能拿它回答 later-start 是否更稳。
  - `exp24` 的 dynamic relative loss 版本已经跑到 `step_8000` 左右；当前看到的是 `step_6000` 一度还行，但 `step_8000` 已明显恶化，说明这版动态 scale 目前不 work，并已停止。
  - `exp26` 已经落盘 `step_2000`，当前为 MHD `1.1631`、GRF `1.0618`；相对 `exp24 step_2000` 的 MHD `7.4550`、GRF `2.4649`，这是当前最强 early positive signal；但该 run 已在约 `step_2400` 手动停止并给 `exp32` 让位，因此 fixed-scale 是否能跨过后续窗口仍待观察。
  - `exp27` 已经落盘 `step_0`，当前为 MHD `1.0110`、GRF `1.0458`；但因为 `rollout_dt=1.0` 同时把评估协议改成 `10 NFE`，所以还不能回答 `rollout_dt=1.0` 是否真的更稳。
  - `exp28` 已经落盘 `step_0`，当前为 MHD `1.8126`、GRF `1.1483`；但 no-detach 只影响训练反传路径，因此还不能回答 no-detach 是否更稳。
  - `exp29` 当前本地已有 run_tag，但因 `No CUDA GPUs are available` 卡在初始评估窗口；因此这一步只能说明 best-teacher 分支已尝试 launch，不能说明它正负结果如何。
  - `exp30` 当前只有用户口头同步 launch；本地尚未看到 run 目录与 log，因此还不能算进入“可比较窗口”。
  - `exp31` 当前已由用户同步在 `20260308215522 : t2` launch；但本地尚未看到 run 目录与 log，因此还不能回答 `soft-Linf` 是否有独立净收益。
  - `exp32` 当前已由用户同步在 `H800 : t1` 所在 session 手动启动；这是以 `exp22` 为底、只关掉 `multi-step PDE` 的 clean ablation，但当前本地尚未看到 run 目录与首轮 eval，因此还不能回答 `multi-step PDE` 是否有独立净收益。
  - `soft-Linf` 与 multi-step PDE 这个组合配置族当前是有净改进的，但两者的独立贡献还没拆清。
- 未回答结果：
  - 每次 `teacher refresh` 之后，模型到底是在变稳，还是只是在更晚的时间点再次爆炸。
  - dynamic relative loss 和 fixed-scale relative loss，哪种更能把训练残差尺度平衡转化成最终 rollout。
  - `teacher` 生成态的“更物理”目前还缺直接证据，只能从 downstream rollout 间接推断。
  - `batch_size` 到底有多大影响，当前还没有 clean ablation。
  - `exp28` 的 no-detach 是否比 `exp22` 的 detach 版本更稳，还是只是把反传链变长并放大训练不稳。
  - ensemble RHS averaging / windowed perturb target 这类 target-smoothing 分支还没有任何实现或实验。

## 结果影响评估

- 这轮结果改变了什么判断：
  - `Q05` 已经从“候选路线之一”变成当前最值得盯的纯 online 主线。
  - 当前最值得继续跟的不是再回到 `learnable GRF`，而是先把 `exp22 / exp23 / exp31 / exp32` 的关键问题回答清楚；`exp24` 可先归为 dynamic target-RMS 失败分支，`exp26` 归为 fixed-scale early-signal 分支；`exp29 / exp30 / exp31 / exp32` 让 `Q05` 多了 teacher source、rollout curriculum、clean no-soft-Linf 与 clean no-multi-step 四个协议层新分支。
  - `exp27`、`exp28` 与 `exp32` 让 `Q05` 多了三个更直接的 clean 分支，分别对应 `dt` 对齐、no-detach 反传路径与 no-multi-step 目标，可以帮助判断当前不稳到底更像时间尺度错位、梯度路径受限，还是局部 rollout 目标本身的重要性。
- 不改变什么判断：
  - anchored baseline 仍是最强 practical 参照。
  - 不能把 `exp22` 的改善写成 strict data-free 已经成立。
  - 不能把某个 loss trick 的收益直接等价成“teacher 生成态已经更物理”。
