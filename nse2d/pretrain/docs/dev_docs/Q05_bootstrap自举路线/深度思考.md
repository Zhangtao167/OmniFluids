# 深度思考

## 元信息

- QID：Q05
- 最近更新时间：2026-03-11
- 当前要回答的核心问题：
  - bootstrap / self-training 的真实收益，到底来自 teacher 生成态更物理，还是只来自更强的局部训练 trick。
  - teacher 该更早上线还是更晚上线。
  - relative PDE loss 改善的是 field balance 本身，还是最终 rollout 稳定性。
  - latest teacher、best teacher 与 rollout curriculum 这些协议层选择，到底哪些会改变 self-training 的稳定性。
  - `rollout_dt` 是否应与 `dt_data` 对齐，还是保留更细的 model substep 更合理。
  - `soft-Linf / multi-step PDE / batch_size` 这几个量当前各自贡献多大，还是只存在组合配置族收益。
  - no-detach 与 ensemble / perturb-target 这两类新分支值不值得继续开。

## 已确认事实

- 事实 1：
  - `exp13` 作为历史 self-training 基线，最终 inference 是 MHD `28.12`、GRF `25.01`，说明“晚启动 + 3-step teacher rollout”的旧方案并没有把路线跑通。
- 事实 2：
  - `exp18` 把 `teacher` 提前到 `30000` 且 rollout 变成 5 步后，提升到 MHD `17.37`、GRF `16.13`；说明更积极的 bootstrap 配置族明显优于 `exp13`。
- 事实 3：
  - `exp22` 在 `exp18` 之上加 `soft-Linf + multi-step PDE` 后，当前汇总提升到 MHD `11.11`、GRF `9.614`，是目前最强的纯 online / bootstrap 结果。
- 事实 4：
  - `exp22` 仍没有解决长期 rollout。最终 inference 上，MHD 的 `U=19.65`、`n=15.74`，GRF 的 `U=26.70`；step-10 时 MHD `U=71.73`、`n=58.93`，GRF `U=94.68`，爆炸非常明显。
- 事实 5：
  - `exp23` 当前已落盘到 `step_18000`，而它要到 `step_100000` 才启动 bootstrap；因此当前所有 `exp23` 结果都还是 pre-bootstrap 观察。
- 事实 6：
  - `exp24` 的 dynamic target-RMS relative loss 版本已经跑到 early-to-mid 观察窗口；`step_6000` 时 MHD `1.875`、GRF `1.199`，但到 `step_8000` 已恶化到 MHD `8.486`、GRF `2.905`，且 `U` 爆炸明显，因此当前已停止继续占算力。
- 事实 7：
  - `exp27` 已在 `H800 : t2-2` 落盘 `step_0`，当前为 MHD `1.0110`、GRF `1.0458`；但因为它把评估协议改成了 `10 NFE`，现在还不能把这个数值直接当正结果。
- 事实 8：
  - `exp28` 已在 `a100_dev : t` 落盘 `step_0`，当前为 MHD `1.8126`、GRF `1.1483`；但 no-detach 只影响训练反传路径，所以这一步只能说明 run 正常，不说明 no-detach 有效。
- 事实 9：

## 核心假设

- 假设 A：
  - bootstrap 有价值的前提，是 teacher 至少已经学到“局部几步内不过分离谱”的动力学；否则 teacher 只会把自己的偏差反复喂回训练输入。
- 假设 B：
  - `exp13 -> exp18 -> exp22` 的收益，不一定主要来自“teacher 更早上线”；也可能更多来自 teacher rollout 更长、局部 loss 更稳、multi-step 约束更强。
- 假设 C：
  - `soft-Linf + multi-step PDE` 更像是在改善“局部 rollout 训练质量”，而不是直接证明 teacher 生成态已经更接近真实物理流形。
- 假设 D：
  - relative PDE loss 解决的是“不同 field 的残差尺度不平衡”；它未必能自动解决 rollout 阶段真正最先爆炸的 field。
- 假设 E：
  - 如果当前不稳的一部分来自 `rollout_dt=0.1` 的内部 substep 误差积累或 model-step / data-step 错位，那么把 `rollout_dt` 直接对齐到 `1.0` 可能会改变失稳模式。

## 反例攻击

- 最脆弱的前提：
  - “只要 self-training 指标继续变好，就说明 teacher 在生成更物理的训练态。”
- 什么现象一旦出现就说明这条路不通：
  - `exp23` 到了 `step_100000` 以后依然不优于 `exp22`，说明“晚一点再开 teacher”本身并不能解决自举放大错误。
  - `exp24` 在 `step_6000 -> 8000` 窗口已经出现明显反转并快速失稳，说明 dynamic target-RMS relative loss 更像是在重标定局部 loss，而没有把收益转成 rollout 改善。
- 目前最强反对意见：
  - 当前看到的收益，可能主要来自 `soft-Linf`、`multi-step PDE` 等局部目标，而不是来自 teacher 真正把输入分布推向更物理的区域。

## 路线比较

| 路线 | 为什么可能成立 | 最大风险 | 当前判断 |
|------|----------------|----------|----------|
| `exp13`：历史 self-training 基线 | 至少验证了 teacher 自举机制可实现 | teacher 太弱、上线太晚、rollout 太短，几乎只是在放大错误 | **已可判定为不够好，只保留作差基线** |
| `exp18`：更早启动、5-step teacher rollout | 让 teacher 更早参与、生成更丰富的训练态 | 与 `exp13` 不是 clean timing ablation，收益来源不纯 | **证明更积极的 bootstrap 配置族更值得继续** |
| `exp22`：`exp18` + `soft-Linf + multi-step PDE` | 同时改善峰值误差控制和局部 autoregressive 稳定性 | 仍可能只是把局部训练做好，而没有解决 teacher 偏差回灌 | **当前主线** |
| `exp23`：晚启动 bootstrap | 如果早期 teacher 太差，晚启动可能避免错误过早自举 | 它同时改了 `update_every=10000`，不是纯时机对照；且当前还没到激活点 | **高优先级运行中，但不能提前写结论** |
| `exp24`：relative PDE loss bootstrap | 可能缓解大数值 field 主导，让优化更关注 field balance | 训练时的尺度平衡不等于 rollout 时的稳定性；当前 early-to-mid 证据已经偏负面 | **当前可暂判不 work，保留作 dynamic-scale 负例** |
| `exp26`：fixed-scale relative loss bootstrap | 如果 dynamic target-RMS 的噪声太大，固定 per-field scale 也许更稳 | 即便 `step_2000` 信号很强，也仍要提防后续窗口重复 `exp24` 的反弹 | **高优先级运行中，当前已拿到最强 early positive signal** |
| `exp27`：model `dt=1.0` bootstrap | 如果内部 substep 或 model-step / data-step 错位是当前不稳来源之一，那么直接对齐可能更稳 | 更粗的模型步长也可能让局部动力学更难学；而且 `step_0` 协议已变成 `10 NFE`，不能直接和 `exp22` 下结论 | **高优先级运行中，但当前只算 very-early 新分支** |
| `exp28`：no-detach bootstrap | 如果当前 detach 砍断了跨步梯度，保留完整反传也许更利于 multi-step 训练 | 也可能只是让训练更不稳、更吃显存；而且 `step_0` 还看不出任何差别 | **高优先级运行中，但当前只算 very-early 新分支** |
| `exp29`：best-teacher bootstrap | 如果 latest teacher 在 refresh 时漂移太大，改用 best checkpoint 也许能减少错误回灌 | 当前还没真正跑起来；如果 launch 都没修好，就无法把它当实验分支比较 | **值得保留，但当前先修 launch，不下方法结论** |
| `exp30`：rollout-curriculum bootstrap | 如果 teacher rollout 太长会过早放大误差，渐进式 curriculum 也许更稳 | schedule 本身会引入额外变量；而且当前本地结果还没落盘 | **已 launch 待落盘，当前不能写结论** |

## 当前判断

- 当前更倾向的路线：
  - 继续把 `exp22` 配置族作为 `Q05` 主线，不回退到更弱的历史 bootstrap 方案。
  - 优先级上，先回答 fixed-scale relative loss 是否比 dynamic scale 更稳，再回答 `teacher` 上线时机；与此同时把 best teacher source 与 rollout curriculum 作为新的协议层 clean 分支保留。
- 暂不推进的路线：
  - 不把 `exp23` 当前 pre-bootstrap 的 `step_0 / 10000` 指标解释成 later-start 已经有效或无效。
  - 不再继续跟 `exp24`；当前已把它归档为 dynamic-scale 负例。
  - 不把 `Q05` 的改善误写成“teacher 已经学会生成物理态”。
