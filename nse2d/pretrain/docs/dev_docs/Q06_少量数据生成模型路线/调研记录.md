## 冲突与缺口

- 证据冲突：
  - 这条路线从“更强状态先验”角度看很相关，但从研究叙事看又已经偏离 strict data-free。
  - 它在概念上可能更合理，但当前无直接实验支持，不能因为概念吸引力就升格为主线。
- 关键缺口：
  - 缺 `Q06` 专属实现、训练脚本、结果表和失败样例。
  - 缺最小真实数据预算定义。
  - 缺需要学习的对象定义：初始状态分布、短 rollout manifold，还是 latent prior。
  - 缺 clean success criterion：到底看样本质量、统计量，还是 downstream surrogate 的 rollout 收益。
- 还需要补什么：
  - 一版严格区分 `strict data-free` 与 `data-informed prior` 的口径说明。
  - 一版 future pilot 的最小设计表：数据预算、模型级别、对照基线、成功阈值。
  - 一版 distribution-level 指标候选：per-field 统计、谱、cross-spectrum、coherence 等。

## 当前结论

- 可直接采用：
  - `Q06` 应保留，但明确写成边界路线 / fallback，而不是当前主攻线。
  - 当前仓库里和 `Q06` 最接近的直接内容，主要是路线讨论、文献占位和更强 prior 的表述；当前无直接实验支持。
  - 当前代码与结果更直接支持的是 `Q04/Q05` 与 anchored baseline 的比较，不应误迁移进 `Q06`。
- 暂不能下结论：
  - 哪种生成模型族最适合当前五场 MHD 状态分布。
  - 多少真实数据才足够，又不会把问题变成普通 supervised pretraining。
  - 这条路线是否真能改善长期 rollout，而不只是生成更“像样”的样本。
