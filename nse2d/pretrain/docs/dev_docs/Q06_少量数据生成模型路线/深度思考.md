## 核心假设

- 假设 A：当前主问题的难点之一确实是状态先验 / 物理流形不够强，因此 `Q06` 在概念上有针对性。
- 假设 B：因为 `Q06` 需要少量真实数据，它会削弱 strict data-free 的研究叙事，所以不应与当前主线混写。
- 假设 C：如果生成态不能转化成 downstream surrogate 的 rollout 改善，那么“样本更像真实状态”本身并不足以构成项目价值。
- 假设 D：如果未来真做 `Q06`，第一步应是低成本、可诊断的 probe，而不是直接上高容量 `diffusion` 级模型。

## 反例攻击

- 最脆弱的前提：
  - “只要从少量真实数据学一个生成模型，就能把训练态推近物理流形，并显著改善 surrogate 训练。”
- 什么现象一旦出现就说明这条路不通：
  - 生成样本在可视化或简单统计上更像真实状态，但 downstream surrogate 的 `MHD/GRF` rollout 仍不优于 fixed `GRF`、learnable `GRF` 或 self-training 基线。
- 目前最强反对意见：
  - `Q06` 可能只是换一种方式重新引入真实状态锚点，从而让研究叙事更像 `data-informed prior`，而不是 strict data-free 的直接突破。

## 路线比较

| 路线 | 为什么可能成立 | 最大风险 | 当前判断 |
|------|----------------|----------|----------|
| 暂不立项，只保留 fallback | 保持主线聚焦在 `Q04/Q05` 与更结构化先验，避免研究叙事漂移 | 可能错过一条最终有效但当前未启动的路线 | 这是当前最合理的选择 |
| 先做低成本 latent prior / `VAE` 风格 probe | 能以较低成本回答“small-data prior 本身有没有信号” | 仍需要先定义数据预算、目标对象和下游成功标准；否则也可能只是学到狭窄 anchor 模仿 | 只有在 gating 问题先回答后才值得考虑 |
| 直接上 `diffusion / latent diffusion / flow` | 表达能力强，理论上更有机会学到复杂状态流形 | 算力和实现成本高，当前又无直接项目证据，失败后也很难判断是模型问题还是设定问题 | 当前不建议直接推进 |
