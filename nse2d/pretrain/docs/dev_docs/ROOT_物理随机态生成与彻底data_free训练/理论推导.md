# 理论推导

## 元信息

- QID：Q01
- 最近更新时间：2026-03-11
- 当前推导目标：
  - 解释为什么“训练 loss 持续下降 + 单步 eval 下降”不等于“多步 rollout 稳定”
  - 形式化当前主问题：如何构造更符合物理的随机训练分布

## 符号表

| 符号 | 含义 | 备注 |
|------|------|------|
| `F` | 真实一步流映射 | 对应 PDE / solver 的一步演化 |
| `G_\theta` | 模型一步流映射 | 参数为 `\theta` |
| `x_k` | 真实第 `k` 步状态 | 在真实轨迹上 |
| `\hat{x}_k` | 模型第 `k` 步 rollout 状态 | 自回归产生 |
| `e_k` | rollout 误差 | `e_k = \hat{x}_k - x_k` |
| `q` | 训练状态分布 | 当前主问题的核心对象 |
| `\mu` | 真实 rollout / attractor 分布 | 真正希望模型表现好的分布 |

## 问题形式化

- 输入：
  - 一个随机生成的训练状态分布 `q`
  - 一个物理训练目标，如 physics loss / PDE residual
  - 一个 surrogate 模型 `G_\theta`
- 输出：
  - 在真实 rollout 分布 `\mu` 上长期稳定的 surrogate model
- 目标：
  - 不只是让 `E_{x~q}[local loss]` 变小
  - 而是让 `G_\theta` 在 `\mu` 上长期 rollout 也稳定
- 约束：
  - 尽量不依赖真实未来轨迹监督
  - 尽量用更物理的随机训练态代替真实状态锚点

## 推导草稿

- 关键步骤 1：为什么单步变好不等于多步变好
  - 若真实系统是 `x_{k+1}=F(x_k)`，模型是 `\hat{x}_{k+1}=G_\theta(\hat{x}_k)`，
  - 则局部线性化下有：
  - `e_{k+1} ≈ J_F(x_k)e_k + (G_\theta(x_k)-F(x_k))`
  - 第二项是局部一步误差，第一项是误差传播
  - 因而即使第二项减小，只要第一项还会持续放大，multi-step rollout 仍会变差

- 关键步骤 2：为什么训练 loss 下降却可能帮不到 rollout
  - 当前训练更像最小化 `E_{x~q}[local residual]`
  - 但评估真正关心的是 `E_{x_0~\mu}[long rollout error]`
  - 若 `q` 与 `\mu` 相差很远，局部目标在 `q` 上的改善无法自动迁移到 `\mu`

- 关键步骤 3：与当前主问题的对应关系
  - strict data-free 的关键不是只找一个 random initializer
  - 而是构造一个更物理的训练分布 `q`
  - 所以后续主线应从“调 loss”转向“构造更符合物理的随机训练态”

## 边界条件

- 适用前提：
  - 模型和真实动力学在局部可近似线性化
  - 当前讨论的爆炸主要发生在 rollout 偏离累积后
- 失效条件：
  - 如果一开始状态就完全脱离物理流形，局部线性化解释会过弱
  - 如果 train / inference preprocessing 差异过大，也会让分布讨论与误差传播混在一起
- 特例：
  - 若 `q ≈ \mu`，则 local residual 的改进更有机会转化成长期 rollout 提升
  - 这也解释了为什么“GRF 预训练 + simulation data 继续训练”目前效果最好

## 未证明点

- 还没打通的地方：
  - 怎样定量衡量“当前随机训练态分布离真实物理流形有多远”
  - 爆炸 pattern 是某几个 field 主导，还是跨 field 联合结构失真主导
- 需要什么额外假设：
  - 对五场联合统计、谱结构、相位关系有更明确的物理先验
  - 对不同 field 是否需要不同约束形式有更系统的经验支持

## 当前结论

- 可用结论：
  - 当前主问题可以形式化为“构造更接近真实物理分布的随机训练分布 `q`”
  - 单步误差下降不等于多步稳定
  - staged / offline rescue 的成功，说明真实流形锚点目前仍然关键
- 暂不成立或待验证结论：
  - “只要 learnable GRF 再调好一点就能解决 strict data-free”
  - “只要 single-step 足够好，multi-step 自然会跟着好”
