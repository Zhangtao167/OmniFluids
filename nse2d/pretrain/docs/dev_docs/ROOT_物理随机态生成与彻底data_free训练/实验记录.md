## 关键结果

- 已确认结果：
  - 当前最好经验路线仍然是“GRF 预训练，然后用 simulation data 继续训练”。
  - raw online 变体中，经常能做到单步误差接近 simulation-data 训练，但多步 rollout 后仍会爆掉。
  - `exp22` 比 `exp18` 有改善，但还远没有解决长期稳定性问题。
- 暂不确定结果：
  - `batch_size` 是主因、次因，还是只是放大器。
  - `n` 与 `Ti` 是否确实不需要绝对值限制，还是只是在当前样本中如此。
  - `exp24` 这版 dynamic relative loss 当前已基本判负；后续还需要看 fixed-scale 分支能否把 loss balance 真正转化成 rollout 稳定性提升。
  - `exp27` 这版 `model dt=1.0` 分支当前已有 `step_0`，但还没有训练后的首轮对照证据。
  - `exp28` 这版 no-detach 分支当前已有 `step_0`，但还没有训练后的首轮对照证据。
  - `exp29` 这版 best-teacher 分支当前先暴露的是 launch / GPU 可见性问题；修好前还不能写方法结论。
  - `exp30` 这版 rollout curriculum 分支当前还没有本地 run 目录与首轮证据。
- 高价值失败：
  - `exp14 (learnable GRF)`：MHD Mean `301.3`
  - `exp13 (GRF self-train)`：MHD Mean `28.12`
  - 这些失败说明：靠当前独立 GRF + trick 组合还不足以得到稳定 rollout

## 结果影响评估

- 这轮结果改变了什么判断：
  - 主问题应写成“更符合物理的随机态生成与彻底 data-free 训练”，而不是泛泛的 strict data-free。
  - `generative model / learnable GRF / self-training bootstrap` 应视为三条候选生成器路线。
  - 当前 `Q05` 的协议层探索已经从 `start / dt / detach / relative loss`，扩展到 `teacher source / rollout curriculum`。
- 不改变什么判断：
  - 真实状态锚点仍然关键。
  - staged 的成功更像是后段 offline 把模型拉回了物理流形。
