# 深度思考

## 元信息

- QID：Q01
- 最近更新时间：2026-03-11
- 当前要回答的核心问题：
  - 为什么 GRF 训练里，训练 loss 和单步 eval 会持续下降，但多步 rollout 会越来越差？
  - 什么样的随机生成训练态才更接近“物理上合理”？

## 已确认事实

- 事实 1：目前效果最好的仍然是“GRF 预训练 + simulation data 继续训练”，而不是彻底 raw online。
- 事实 2：在多个 GRF / online 实验里，单步误差下降和多步稳定性提升并不同步。
- 事实 3：当前 raw online 强化线已经试过：
  - learnable GRF
  - self-training bootstrap
  - soft-Linf
  - multi-step PDE
  - 但还没有根本解决多步爆炸。
- 事实 4：当前独立 per-field GRF 缺少五场联合结构。

## 核心假设

- 假设 A：问题的核心不是“loss 够不够强”，而是当前随机训练态分布不够物理。
- 假设 B：训练 loss 和单步 eval 的持续下降，说明模型确实在学局部映射；但长期 rollout 爆炸说明它没有学到稳定的 rollout 轨道。
- 假设 C：`batch_size` 不只是优化超参数，也影响 online 模式下一步里看到的 GRF 多样性、梯度方差和 learnable GRF 的 Monte Carlo 稳定性。
- 假设 D：随机态生成约束应当 field-specific；当前观察里，`n` 场和 `Ti` 场可能不需要绝对值限制，不应把所有 field 一刀切地施加相同约束。

## 反例攻击

- 最脆弱的前提：
  - “只要把 GRF 参数继续调好，strict data-free 就能自然成功。”
- 什么现象一旦出现就说明这条路不通：
  - learnable GRF、self-training bootstrap、multi-step PDE 全部继续改善 single-step，但 multi-step 依旧系统性爆炸。
- 目前最强反对意见：
  - 问题也可能不主要在初值先验，而在 train/inference mismatch、误差传播和 preprocessing 不一致。

## 路线比较

| 路线 | 为什么可能成立 | 最大风险 | 当前判断 |
|------|----------------|----------|----------|
| 生成模型 / 更强随机态先验 | 直接攻击主问题本身，可能给出更接近物理流形的训练态 | 实现成本高，评价不干净时难判断成败 | **当前最值得投入的主线** |
| Learnable GRF | 成本低、改动小、可做解析可控先验 | 只有极低维参数，可能远不足以逼近五场联合结构 | **保留为 baseline / bridge** |
| Self-training bootstrap | 用当前模型把随机态推进到更接近物理流形的位置 | teacher 本身不够强时容易把错误模式自举放大 | **可继续试，但不是终局答案** |
| 继续堆 soft-Linf / multi-step 等 loss trick | 实现快，容易出数 | 如果训练态分布本身错了，可能只是在错分布上做更复杂优化 | **只能作为辅助手段** |

## 当前判断

- 当前更倾向的路线：
  - 主问题应明确成“更符合物理的随机态生成”。
  - 技术上优先顺序应是：
    - 生成模型 / 更强先验
    - learnable GRF
    - self-training bootstrap
- 暂不推进的路线：
  - 不再把“继续在独立 GRF 上叠更多 loss trick”当成主力方向。
