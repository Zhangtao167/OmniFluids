# 主问题文件夹说明

每个主问题使用一个独立目录：

`Qxx_问题短名/`

例如：

`Q01_问题短名/`

## 目的

1. 隔离不同问题的上下文
2. 给 Agent 一个稳定的二级工作区
3. 防止调研、实验、理论、讨论互相污染

## 最小要求

每个主问题至少应有：

1. `README.md`
2. `sub_main.md`
3. 一个稳定的 `QID`
4. 在 `main.md` 主问题总表中有对应行

## `sub_main.md` 的作用

`sub_main.md` 是主问题目录下的二级 `main` 控制面。

它主要负责：

1. 管理当前主问题下的关键子问题拆解
2. 区分问题类型：调研 / 深度思考 / 实验验证 / 理论推导 / Agent讨论 / 老师或同事讨论
3. 区分状态：提出待分析 / 待研究 / 进行中 / 待验收 / 已完成
4. 维护当前运行实验或当前计划表
5. 指向 `实验记录.md`、`调研记录.md`、`深度思考.md` 等具体文档，而不是重复长正文

## 文件触发器

### `调研记录.md`

出现下面任一情况时创建或启用：

1. 需要外部资料、baseline、benchmark、相关工作
2. 需要整理检索结果或证据冲突
3. 需要输出对当前问题有决策价值的调研结论

### `实验记录.md`

出现下面任一情况时创建或启用：

1. 开始写代码验证
2. 开始跑 benchmark 或对比实验
3. bug、重构、配置变体会影响结果判断

### `深度思考.md`

出现下面任一情况时创建或启用：

1. 存在两条及以上候选路线
2. 关键假设尚未定
3. 需要做第一性原理拆解、反例攻击、路线比较

### `理论推导.md`

出现下面任一情况时创建或启用：

1. 出现符号、公式、命题、形式化目标
2. 需要做正式推导或分析边界条件

### `交流讨论.md`

出现下面任一情况时创建或启用：

1. 与 Agent、人类、老师、同事的讨论改变了判断
2. 外部反馈需要记录为后续动作

## `Qxx/README.md` 何时必须回写

仅当下面三类内容变化时强制回写：

1. 状态变更
2. 关键结论变更
3. 最大阻塞变更

## `Qxx/sub_main.md` 何时必须回写

仅当下面四类内容变化时强制回写：

1. 子问题拆解变化
2. 状态迁移变化
3. 运行实验或当前计划变化
4. 已确认结论与待补证边界变化

## `runs/` 何时启用

默认不启用单独 run 文档。

建议在下面情况出现时启用：

1. `run_id` 超过 10 个
2. 多机器、多分支、多配置并行
3. 高价值失败 run 需要完整保留

## `assets/` 存什么

1. 图表
2. 表格截图
3. 草图
4. 日志截图
5. 讨论白板或示意图

## 反模式

1. 把长篇正文直接堆到 `main.md`
2. 把只属于本问题的内容写进 `shared_context/`
3. 没有 `run_id` 的实验记录
4. 更新过程文件但不回写 `sub_main.md`、`README.md` 和 `main.md`
