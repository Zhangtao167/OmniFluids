    1. 实验管理开发文档
      1. 原则
        1. 必须记录，无记录不实验
        2. 必须尽可能用最快的迭代速度
        3. 必须有passion （价值认同 or 兴趣驱动）
        4. Agent native 流程
      2. 文件结构
        1. dev_docs （做深度实验的agent协作开发文档，用来管理我开发迭代深度学习算法的文档系统。几个特点问题驱动式；Agent native；结构化层次化文档；更多记录）
          1. 构建文档开发文档系统预问题和答案（主要是使用这个模版的一开始要回答的问题，和agent交互格式构建起这个文档模版，问题可以使选择、简答题、补充说明等，最好是选择（方便交互））
          2. main.md  （注意这个文档主要放大纲，结论等。不要太冗余，具体内容指向具体的文档）
            1. 目标
              1. 根问题
              2. 验收指标
            2. 资源和工具
              1. 算力
              2. 人力
              3. 工具
            3. 研究的问题（主问题、子问题；））
              1. 主问题1-XXXX
                1. 已有的结论和观察
                2. 子问题1
                3. 子问题2
              2. 主问题2-XXX
                1. 。。。
              3. 这个部分具体注意如下（不是正文）
                1. 区分问题类型
                  1. 需要调研的问题
                  2. 需要深度思考的问题
                  3. 需要实验验证的问题（写code实现，然后实验验证）
                  4. 需要理论推导的问题
                  5. 需要和agent讨论的
                  6. 需要和老师、同事交流讨论的
                2. 分别标记状态
                  1. 提出待分析（需要进一步第一性原理分析、调研、深度思考和决策的）
                  2. 待研究（已经经过分析，有了验收指标）
                  3. 进行中 （正在做实验、调研、深度思考、理论推导、）
                    1. 这是主体，我和agent每次关注、更新维护这里
                    2. 会把正在运行的实验搞成表格放在这里，每条实验记录要包括code git commit,实验目的，实验机器（哪台机器什么tmux session）、结果文件位置、预期结果和验收条件等、实验结果、分析和下一步计划
                    3. 调研计划和过程，并指向一个调研报告文档
                    4. 深度思考的过程
                    5. 理论推导的过程
                    6. 交流讨论过程
                  4. 待验收
                  5. 已完成
                    1. 每次我或者agent（主要是我）将待验收的问题验收后会将实验、调研、深度思考、理论推导的过程整理到对应具体文档去。这里只放总结的论题，结论和论据概述，并指向具体的细节文档。
