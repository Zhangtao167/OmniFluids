## 已确认事实

- 事实 1：当前效果最好的经验路线仍带真实状态锚点；strict data-free 还没有成立。
- 事实 2：`exp18 / exp20 / exp21 / exp22` 这类 raw online 强化线都说明，single-step 改善不自动转化成长期 rollout 稳定。
- 事实 3：`exp_offline_pde (1GPU)` 与 `exp1b (sup dt=1s, 1GPU)` 都在稳定可比较区间，说明模型并不是完全学不会 physics / one-step map。
- 事实 4：当前独立 per-field GRF 缺少五场联合结构；按项目稳定共识，整体更接近 `Level B/C` 而非 strict `Level D`。

## 核心假设

- 假设 A：当前主矛盾首先是训练分布支持问题，即随机训练态 $q$ 与真实 rollout 分布 $\mu$ 相差过远，而不是 `PDE loss` 本身不可学。
- 假设 B：staged / offline anchor 当前最好，主要不是因为 warmup trick 本身神奇，而是因为真实状态锚点把训练支持集拉回了更接近物理流形的区域。
- 假设 C：one-step 与 rollout 的分裂来自两个因素的耦合：局部一步目标与长期稳定目标不一致，以及误差在自回归 rollout 中持续放大。
- 假设 D：更物理的先验应当是 joint / field-specific 的；`n` 和 `Ti` 是否需要绝对值限制仍需专门验证，不应一刀切。

## 反例攻击

- 最脆弱的前提：
  - 把“分布错配”直接当成唯一根因。
- 什么现象一旦出现就说明这条路不通：
  - 在 train/inference 协议清理后，且训练态已明显更接近真实联合结构时，multi-step rollout 仍与 single-step 完全脱钩。
  - 用更强 joint prior 之后，结果仍不优于 anchored baseline 的近邻参照。
- 目前最强反对意见：
  - 问题也可能同样强地来自 train/inference mismatch、preprocessing 差异、val/test 混用，以及误差传播本身，而不是先验单点失真。

## 路线比较

| 路线 | 为什么可能成立 | 最大风险 | 当前判断 |
|------|----------------|----------|----------|
| anchored baseline / staged rescue | 最接近真实状态支持，能给出当前最可信的上界与参照系 | 不是 strict data-free，不能当作根问题已解决 | 必须保留为参照，但不能误当终点 |
| fixed random prior / PDE-only | 是最干净的 strict data-free 对照，能直接暴露先验上限 | 若训练分布离真实 rollout 分布太远，长期 rollout 可能天然不稳 | 必须保留为基线与下界，不应独自承担全部希望 |
| learnable GRF | 成本低、可解析、能验证“可学习先验”是否有增益 | 参数化太弱，可能只改善局部指标，不足以恢复五场联合结构 | 适合作为 bridge / baseline，不宜仅凭局部收益升格为主线 |
| self-training / bootstrap | teacher 可能把训练态逐步推向更物理、更稳定的区域 | teacher 会放大自身错误模式，收益可能在 refresh 后塌缩 | 目前最值得继续跟踪的 pure online 候选线 |
| stronger joint prior / generative model | 直接攻击“随机态不够物理”这个根因 | 实现成本高，且若协议不干净很难解释成败 | 从第一性原理看最值得长期投入，但当前证据仍不足 |
| 继续堆 soft-Linf / multi-step 等 loss trick | 实现快，能改善局部指标或训练稳定性 | 若训练支持集错了，容易只是在错分布上优化得更努力 | 只能作为辅助手段，不能替代更强先验 |

## 当前判断

- 当前更倾向的路线：
  - 这个问题有明确价值，因为它直接决定能否在复杂 PDE surrogate 训练中显著减少对真实仿真轨迹的依赖。
  - 按当前项目内口径，`Level B/C` 可行性已经有一定支撑：带真实状态锚点的 physics-only / supervised baseline 都能稳定工作。
  - strict `Level D` 仍未证成：现有随机态-only 路线还没有在可信多步 rollout 上成立。
  - 在方法投入顺序上，更强先验 / joint prior 与 bootstrap 比继续堆 loss trick 更值得优先投入；learnable GRF 适合保留为 bridge。
- 暂不推进的路线：
  - 不把 staged / offline rescue 的成功写成 strict data-free 已解决。
  - 不再把“再叠一个 loss trick”当成 Q02 的主回答。
