  - 形式化解释为什么 `train loss` 下降和 one-step 指标改善，不等于 multi-step rollout 稳定。

## 符号表

- $F$：真实一步流映射，对应 PDE / solver 的一步演化。
- $G_\theta$：模型一步流映射，参数为 $\theta$。
- $x_k$：真实第 $k$ 步状态，在真实轨迹上。
- $\hat{x}_k$：模型 rollout 的第 $k$ 步状态，由自回归产生。
- $e_k$：rollout 误差，定义为

$$
e_k = \hat{x}_k - x_k.
$$

- $q$：训练状态分布，是当前真正可控的对象。
- $\mu$：真实 rollout / attractor 分布，是模型真正希望表现好的分布。
- $\ell$：局部一步损失或 residual，可对应 PDE loss / local loss。
- $J_F$：真实动力学的局部雅可比，用于线性化误差传播。

## 问题形式化

- 输入：
  - 一个随机生成训练分布 $q$
  - 一个局部训练目标，如 physics loss / PDE residual
  - 一个 surrogate 模型 $G_\theta$
- 输出：
  - 在真实 rollout 分布 $\mu$ 上长期稳定的 surrogate model
- 目标：
  - 优化

$$
L_q(\theta) = \mathbb{E}_{x \sim q}\bigl[\ell(G_\theta(x), F(x))\bigr].
$$

  - 同时希望

$$
R_\mu^{(T)}(\theta)
= \mathbb{E}_{x_0 \sim \mu}\left[\sum_{k=1}^{T} \lVert \hat{x}_k - x_k \rVert \right]
$$

    保持可控。
- 约束：
  - 尽量不依赖真实未来轨迹监督
  - 尽量减少真实状态锚点
  - train/inference path、时间尺度、val/test split 需要尽量一致

## 推导草稿

- 关键步骤 1：局部误差下降为何不保证长期稳定
  - 若真实系统满足 $x_{k+1} = F(x_k)$，模型 rollout 满足 $\hat{x}_{k+1} = G_\theta(\hat{x}_k)$，
  - 局部线性化可写成

$$
e_{k+1} \approx J_F(x_k)e_k + \bigl(G_\theta(x_k) - F(x_k)\bigr).
$$

  - 第二项是一步逼近误差，第一项是误差传播。
  - 因而即使局部一步误差下降，只要传播项持续放大，`step-10` 与 `Mean` 仍可能变差。

- 关键步骤 2：为什么在 $q$ 上学得更好，不代表在 $\mu$ 上 rollout 更好
  - 当前训练更像最小化 $L_q(\theta)$，但评估真正关心的是 $R_\mu^{(T)}(\theta)$。
  - 若 $q$ 与 $\mu$ 的支持集差异很大，在 $q$ 上优化到更小 residual，未必能迁移到 $\mu$ 上的自回归 rollout。

- 关键步骤 3：为什么真实状态锚点当前仍关键
  - anchored / staged 路线的作用，可理解为把训练支持集拉回更接近 $\mu$ 的区域。
  - 这解释了为什么 `exp_offline_pde (1GPU)` 这类 physics-only anchored baseline 是可行的，而随机态-only online 线仍不稳。

- 关键步骤 4：Q02 的可行性判断
  - 若 `PDE loss` 在 anchored support 上可学，而在 random support 上失效，则障碍更像是 $q \to \mu$ 的构造问题，而不是“physics objective 不可学”。
  - 因此当前可行性的核心不在于再加局部 loss，而在于让随机训练态更接近真实物理流形，并降低 protocol mismatch 对判断的污染。

## 边界条件

- 适用前提：
  - 当前爆炸主要发生在误差积累后的 rollout 阶段。
  - 模型与真实动力学在局部可近似线性化。
- 失效条件：
  - 如果 preprocessing / normalization / time-scale mismatch 主导误差，单靠 $q$ vs $\mu$ 分析不足以解释全部现象。
  - 如果初始随机态一开始就完全脱离物理流形，局部线性化只提供很弱解释。
- 特例：
  - 若 $q \approx \mu$，局部 objective 的改善更有机会迁移为长期 rollout 提升。
  - `mhd5_staged_v1` 的 MHD 强、GRF 差，提示“离线救回”与“strict data-free 成立”是两件不同的事。

## 未证明点

- 还没打通的地方：
  - 如何量化五场联合结构下 $q$ 与 $\mu$ 的距离。
  - `batch_size`、joint structure 缺失、field-specific constraints 三者各自贡献多大。
- 需要什么额外假设：
  - strict `data-free` 允许哪些先验的正式定义。
  - `Level B/C/D` 的正式定义。
  - “最小充分先验”需要满足哪些联合统计、频谱或相位关系。

## 当前结论

- 可用结论：
  - Q02 可以形式化为：在不依赖真实未来轨迹监督的前提下，构造足够接近真实 rollout 分布 $\mu$ 的训练分布 $q$。
  - one-step 指标改善不等于长期 rollout 稳定。
  - 当前 anchored baseline 的成功说明 physics-only 学习并非不可行，但真实状态锚点仍在为分布支持纠偏。
- 暂不成立或待验证结论：
  - “只要 learnable GRF 再调好一点，strict data-free 就能成立”。
  - “只要 `step-1` 足够好，`step-10` 自然会跟着好”。
  - “当前 staged 最优已经证明 random prior warmup 本身学到了正确物理”。
