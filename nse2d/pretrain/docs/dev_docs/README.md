# dev_docs Template

这是一个轻量、问题驱动、Agent Native 的协作文档模板。

它服务四件事：

1. 把项目当前活跃面稳定放在 `main.md`
2. 把不同主问题隔离到各自的 `Qxx_问题短名/`
3. 让实验、思考、理论、讨论都有明确归宿
4. 让关键结论能稳定回写，而不是只留在聊天里

本模板以 `init_design_prompt.md` 为上游设计说明，但这里提供的是可直接复制到项目仓库中的实际文件系统。

## 硬规则

1. `main.md` 只写当前活跃面，不做总档案馆。
2. 在 `main.md` 中，任何单条记录超过 8 到 12 行，默认下沉到对应 `Qxx/`。
3. 一个主问题一个文件夹，不混写。
4. `Qxx/README.md` 负责问题摘要、边界和入口，不承担完整推进控制。
5. `Qxx/sub_main.md` 是每个主问题的二级控制面，用来管理子问题拆解、状态迁移和当前推进。
6. 其余问题文件按需创建或启用，不强制预建空壳。
7. 每次有实质推进后，按这个顺序回写：
   - 具体记录文件
   - `Qxx/sub_main.md`
   - `Qxx/README.md`
   - `main.md`
8. `shared_context/` 只放跨问题可复用的信息；所有共享文件都应登记到 `shared_context/index.md`。

## 目录说明

- `00_init_answers.md`
  - 新项目初始化问答与默认约束
- `main.md`
  - 整个项目的当前活跃控制面
- `template_list/`
  - 主问题文件夹说明与真实模板
- `shared_context/`
  - 跨问题复用的背景、资源、代码地图、共识
- `progress_snapshots/`
  - 周更、阶段切换、重大结论或重大失败时的低频快照
- `meta/`
  - 模板系统本身的改动和使用反馈
- `Qxx_问题短名/`
  - 单个主问题的完整工作区
  - 其中 `README.md` 管摘要入口，`sub_main.md` 管回答推进

## 创建主问题的最小要求

新主问题至少要有：

1. 一个稳定编号，例如 `Q01`
2. 一个问题文件夹，例如 `Q01_问题短名/`
3. 一个入口文件 `Q01_问题短名/README.md`
4. 一个推进控制面文件 `Q01_问题短名/sub_main.md`
5. 在 `main.md` 主问题总表中新增一行

其余文件根据触发器创建，详见 `template_list/主问题文件夹说明.md`。

## 最常见失控点

1. `main.md` 重新膨胀成巨型工作台
2. 把只属于单个问题的内容写进 `shared_context/`
3. 实验没有稳定 `run_id`
4. 更新了具体记录，但没有回写 `Qxx/sub_main.md`、`Qxx/README.md` 和 `main.md`

## 默认建议

1. `Qxx/README.md` 和 `Qxx/sub_main.md` 默认都应建立
2. `sub_main.md` 只做控制面，不复制长正文证据
3. `实验记录.md` 顶部先维护 run 表，再写关键 run 正文
4. `runs/` 只在 run 变多、分支变多、或高价值失败需要保留时启用
